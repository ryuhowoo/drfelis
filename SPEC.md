# 프로모션 애널리틱스 (드르펠리스 사내 AI MD)

> 자사몰 프로모션의 **매출 기여도를 측정 → 유사 프로모션 매출을 예측 → 목표 매출을 위한 프로모션을 처방**하는 사내 전용 도구.

## 1. 목적
"이번 프로모션이 실제로 얼마를 더 벌어줬나"를 데이터로 측정하고, 그 축적된 사례를 근거로
"비슷한 시즌엔 A보다 B가 매출에 유리하다", "이만큼 팔려면 이런 혜택을 걸어라"까지 제안하는
초전문 MD 역할의 분석 서비스.

## 2. 데이터 입력

### 입력 ① 일별 매출 추이 (baseline 엔진의 연료)
| 컬럼 | 예시 |
|---|---|
| 일자 (일) | 2026-02-04 |
| 기초 상품명 | (제품) 플레이 레이지 파자마 L |
| 옵션정보 | 상품선택=PLAY LAZY 파자마 L+퍼펙트 본품 |
| 결제금액 (환불/취소 제외) | 1328967 |
| 기초 상품 판매 수량 (환불/취소 제외) | 19 |

- 약 1년치 일 단위 시계열(예: 365일 × 65개 기초상품).

### 입력 ② 프로모션 시트 (프로모션 기간 실적, 전 제품 포함)
| 컬럼 | 의미 |
|---|---|
| 일자 (누적) | 프로모션 기간 (예: 2025-12-08 ~ 2025-12-14) |
| 기초 상품명 / 옵션정보 | 판매 단위 |
| 결제금액, 결제 건수, AOV | 실적 |
| 수수료, 원가 | 공헌이익 계산용 |
| 기초 상품 판매 수량 | 수량 |
| (선택) Δ%·이전·Δ 컬럼 | 분석툴 기간 비교 익스포트 |

### 마스터 데이터
- **품목코드**: DR코드, 제품원가(VAT±), 소비자가, 상시가
- **자사몰 상품코드**: P코드, 자사몰 리스팅/옵션
- **수량계산**: 번들/옵션 → 기초상품 분해 (`[2+2]` = 기초SKU 4개)

### 사용자가 직접 지정하는 메타데이터 (시트에 없음)
- **메인 상품**: 특별 혜택을 부여한 헤드라인 상품
- **혜택 구조**: 할인율/할인액, 사은품(가치·메인과의 관련성), 1+1/2+2 등
- **목적·시점**: 예) "3주년 세일", "세계 고양이의 날", "신제품 런칭", "리뉴얼 런칭"

## 3. 측정 엔진 (핵심)

```
기초상품별 baseline 일평균
  = 프로모션 직전 8주(56일) 중 "비프로모션 일자"의 일평균 매출/수량
  - 비프로모션 일자 = 어떤 프로모션 기간에도 속하지 않는 날
  - 시즌성은 캘린더 보정(시즌/이벤트 계수)으로 별도 반영

증분(uplift) = (프로모션 기간 실제매출) − (baseline 일평균 × 프로모션 일수)

직접효과 = 메인상품들의 증분 합
후광효과 = 비메인(기타)상품들의 증분 합
프로모션 총기여 = 직접효과 + 후광효과
```

- **분석 단위**: 기초상품(SKU) 단위로 증분 측정 → 옵션구성·프로모션 단위로 성과 롤업.
  (번들은 평소 baseline이 없으므로 측정은 기초상품, 해석은 옵션/프로모션 레벨.)
- **메인 vs 기타**: v1은 메인=사용자 지정, 그 외 전부=기타(후광). 비메인 제품의 상시 할인은 v1에서 무시.

### 성과 지표 (프로모션 단위)
- 총 증분 매출
- 증분율(uplift %) = 증분 / baseline
- 일평균 증분
- 후광 비중 = 후광효과 / 총기여
- 혜택 1원당 증분 (= 증분 / 총 할인액)
- 공헌이익 / 공헌이익률

## 4. 예측 엔진 (방식 A — 규칙/통계 기반)
- 각 프로모션을 **특징 벡터**로 표현: 혜택종류 · 할인율/할인액 · 사은품가치/관련성 · 기간 · 대상 카테고리 · 시점(시즌·이벤트) · 목적.
- 신규(계획) 프로모션 입력 → **유사 과거 프로모션 검색** → 시즌·캘린더 보정 → **예상 매출/증분** 추산.
- **콜드스타트**: 사례가 적으면 카테고리 평균·휴리스틱으로 fallback + **신뢰도(confidence) 명시**.

## 5. 처방 엔진
- "이만큼 팔고 싶다"(목표 매출) 입력 → 혜택 구성 후보 탐색 → **목표를 맞추는 프로모션 형태 추천**.
- 매출 증분뿐 아니라 **공헌이익**까지 함께 랭킹.

## 6. "집요하게 묻기" (제품 기능)
- 실적이 예측과 어긋나거나 유독 튀면, 시스템이 **정성 원인을 캐묻고 기록**:
  광고 증액? 인플루언서 언급? 경쟁사 품절? 시즌 특수? 신제품 효과?
- 이 정성 메모/가설이 **다음 예측·해설의 근거로 축적**됨.

## 7. 사례 탐색 (그 자체로 가치)
- 과거 프로모션을 **유형/특징/성과 기준으로 필터·정렬·비교**하는 라이브러리 뷰.

## 8. 인증 / 배포
- **구글 로그인 + `@drfelis.com` 도메인 제한** (사내 누구나 사용).
- 스택: **Next.js + Supabase(인증·DB) + Vercel**.

## 9. 단계적 범위 (MVP → 확장)
1. **Phase 1 — 측정 + 사례 탐색**: 데이터 업로드/파싱, baseline·증분 계산, 프로모션별 성과 대시보드, 사례 라이브러리.
2. **Phase 2 — 예측**: 특징 벡터·유사 프로모션 검색, 예상 매출 추산, 신뢰도 표기, 정성 메모 수집.
3. **Phase 3 — 처방**: 목표 매출 → 혜택 구성 추천, 공헌이익 랭킹.

## 10. 확정된 설계 결정 (의사결정 로그)
- 기여도 = 증분(uplift), baseline = 직전 8주 비프로모션 일평균(+캘린더 보정).
- 데이터 단위 = 일별 × 기초상품, 롤업 = 옵션/프로모션.
- 후광효과(동반구매)를 총기여에 포함.
- 관점 = 공격적(혜택→성과 매핑). ROI/공헌이익은 참고 지표로 병기.
- 예측 엔진 = 규칙/통계(A), 사례 적어도 답을 내되 신뢰도 명시.
- 입력 = 엑셀/CSV 업로드(일별 매출 추이 + 프로모션 시트 + 마스터).
