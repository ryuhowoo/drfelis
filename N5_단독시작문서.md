## 4. 이번 작업의 본질 — 확정 결정 번복

### 4.1 폐기 선언

> **기존 의사결정 “promotions ─1:1─ campaign_plans”(S0 §2-3)를 폐기한다. 플랜은 promotions에 강결합되지 않는 독립 아티팩트다.**

### 4.2 왜 (사용자가 발견한 문제)

플랜과 실적 시트가 서로 다른 코드로 업로드돼 별개 `promotions` 행이 생기고, 머지 시 “양쪽 모두 플랜 보유”로 거부됨. 근본 원인: 캠페인 상세의 **‘플랜 만들기’ 버튼 클릭만으로 빈 draft 플랜이 자동 생성**(급소 2). 사용자는 플랜 수동 입력 기능을 요청한 적 없음. → **플랜은 플랜대로, 실적은 실적대로 적층. 비교는 명시적 1:1 짝짓기로만.**

### 4.3 N4.3 브랜치 폐기

`claude/n4-3-plan-actual-link` 브랜치(마이그레이션 0020: `campaign_plans.actual_promotion_id` + redirect 함수들)는 **PR 미생성 상태 그대로 폐기**한다. 같은 방향이지만 plan→promotion 의존을 못 벗어났다. 단 “플랜이 비교 대상 실적을 명시”하는 아이디어는 §6.1의 짝짓기 링크로 계승.

### 4.4 마이그레이션 원칙 예외

“additive only, drop 금지” 원칙에서 이번만 예외: `campaign_plans.promotion_id`를 **drop하지 않고 nullable 전환** + 자체 식별자 컬럼 추가로 파괴 최소화.

## 5. 확정 설계 (전 항목 사용자 검증 완료)

샘플: 실적 `CF_P_260518_벌크UP`(2026-05-18~19) × 플랜 `CF_P_260511_벌크UP`. 두 캠페인은 같은 행사의 실적/플랜이다.

### 5.1 달성률 지표 — 매출 0순위

|층     |지표        |산식                        |위상                     |
|------|----------|--------------------------|-----------------------|
|**L0**|**매출 달성률**|Σ실적매출(계획분) / 플랜 목표매출      |**0순위 KPI** (캠페인+옵션 단위)|
|L1    |옵션 세트 달성률 |실적 `판매 수량`(C13) / 플랜 예상세트수|보조·드릴다운(“왜 빗나갔나”)      |
|~L2~  |~SKU 달성률~ |—                         |**v1 보류**              |
|CM    |공헌이익 달성률  |(X − 차감) / 플랜 공헌이익        |참고 지표                  |

- 🔴 **L1 분자는 반드시 실적 C13 `판매 수량`(세트 단위).** C12 `기초 상품 판매 수량`은 번들 분해값(×8 등)이라 쓰면 달성률 400%로 부풀어진다. 둘은 167행 중 91행에서 다르다.

### 5.2 공헌이익 차감 모델

실적 시트엔 **물류비·적립금 컬럼이 없고 광고비가 행 단위로 거의 0**(167행 중 163행) → 행 단위 직접 산출은 부정확. 대신:

```
계획분 실적 공헌이익
 = X (사용자 수기 입력: 캠페인 전체 실적 공헌이익, 회계값)
   − 정기구독 공헌이익     = Σ(정기구독 옵션 판매수량 × 정기구독 단가표)
   − 계획외 일반 "입력" 공헌이익 = Σ(판매수량 × 옵션별 정의액)

공헌이익 달성률 = 계획분 실적 공헌이익 / 플랜 공헌이익(폼값)
```

근거: 회계 X는 광고비가 실제 배분된 정확값. 근사를 큰 덩어리(계획분 86.7%)가 아닌 작은 덩어리(차감분 ~13%)에만 싣는다.

### 5.3 매출 4-bucket 분류

|버킷            |정의                                    |처리                            |
|--------------|--------------------------------------|------------------------------|
|**계획분**       |플랜 옵션에 매칭되는 실적 행                      |L0·L1·CM 분자                   |
|**정기구독**      |상품명에 `[정기구독] 모래` / `[정기구독] 영양케어`      |CM 단가표 차감. 매출은 별도 카드          |
|**계획외 일반(입력)**|그 외 행 중 공헌이익액 정의됨                     |CM 정의액 차감. 매출은 계획외 카드         |
|**계획외 일반(제외)**|`[첫구매]`·`혜택 골라담기`·`[벳트너스]` 증정·기타 제외 분류|X에서 통째 제외(공헌이익≈0 가정). 어디에도 안 셈|

확정: **첫구매 = 제외** / **포캣네스트 커버 = 제외** / 제외 행의 공헌이익은 0으로 가정(증정·손실리더).

### 5.4 ★ 날짜 분기 (정기구독)

2026-01-12 정기구독 개편(간식 삭제, 02-11까지 유예 판매):

```
캠페인 종료일 <= '2026-02-11':
  정기구독 차감하지 않음. 달성률에 그대로 포함(분자 overflow 허용).
  정기구독 매출만 별도 표기. (개편 전 구조라 단가표 매핑 불가)
캠페인 종료일 >= '2026-02-12':
  정기구독 공헌이익을 단가표로 산출해 X에서 차감. 제외 버킷은 X에서 제외.
```

### 5.5 옵션 매칭 키 (검증: 10옵션 중 9 자동 매칭)

정규화 파이프라인 (실적 `옵션정보` → 키):

1. `상품선택=` / `상품선택1=` 접두어 제거
1. **번들 태그 `[n+n]` 제거** (확정 — `[4+4]` vs `[1+1+1]` 형태 불일치도 잡기 위함)
1. 할인율·장식 제거: `[35%⬇️]`, `❤️미야옹철 추천❤️`, `💛집사 최다 구매💛` 등
1. 브랜드 접두어 제거: `카사벤토` `벤토나이트` `퓨저나이트`
1. 공백·구두점 정규화 → **`제품명 + 용량`** 키 (예: `원티드 4.3kg`)

폴백 3단: ①완전일치 → ②부분 키 후보 제시 → ③**수동 매핑 큐**(0017 인프라를 옵션 매핑으로 확장).
**플랜 1옵션 ↔ 실적 N행 합산 허용** (`[4+4][순차출고]`, `[6/4 순차출고]` 분산 대비).
기존 `normalize_sku_name`(0018)을 옵션 레벨로 확장 재사용.

### 5.6 공헌이익액 결측 처리

- 정기구독 단가표의 빈 칸 = **“해당 옵션 없음”**(결측 아님). 0 처리 금지.
- 실적에 판매됐는데 공헌이익액이 없는 옵션 = **“공헌이익액 입력 필요” 배지** + 수기 입력 UI. (샘플에선 정기구독 프로바이오틱스 1박스 1건)

## 12. 사전 정리 SQL (N5.0에서 실행)

```sql
-- 빈 draft 플랜 확인 (옵션 0개) → 확인 후 delete
select cp.id, cp.promotion_id, cp.status, p.name
from promo.campaign_plans cp
join promo.promotions p on p.id = cp.promotion_id
left join promo.campaign_plan_options cpo on cpo.campaign_plan_id = cp.id
where cp.status = 'draft'
group by cp.id, cp.promotion_id, cp.status, p.name
having count(cpo.id) = 0;
-- confirmed 플랜 건수도 확인 (0건 예상 — 0이 아니면 보고 후 진행)
select count(*) from promo.campaign_plans where status='confirmed';
```
